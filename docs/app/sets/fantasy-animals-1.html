セット別設計書: 架空のかわいい動物図鑑

作成中（12-1 で投稿先アカウント・キャプションを確定） set_code: fantasy-animals-1 最終更新: 2026-07-18
本書の位置づけ 本セット固有の設定・運用情報を記録する（11-1 で作成）。共通の処理フロー・データモデル・運用ルールは batch-flow.html / data-model.html / operation.html を参照し、本書には差分のみ書く（design-outline.html セクション 1.1）。

1. テーマ

現実に存在していそうで実在しない、架空のかわいい動物を紹介する画像を生成する。野生の姿を偶然写真に収めたかのような自然な写真風の構図とし、画像内に架空の動物の名前とその特徴（外見・生態など）の説明文を記載する。

2. 投稿先アカウント構成

未定（12-1 で sns_accounts と SNS 認証 Secret を登録して確定する）。プラットフォームは Instagram を想定（初期スコープ）。

3. スケジュール（投稿頻度）

未定。初セットは既存プレースホルダ Scheduler（acps-prod-image-generation-schedule）を転用する方針（operation.html セクション 2.1 手順 3 の decision）。cron 式の確定・有効化のタイミングは別途判断する（11-1 の完了条件には含まれない）。

4. プロンプトの作り方

固定文言方式。prompt_configs.prompt_text への直接 INSERT で管理する（初期運用は 1 セット 1 prompt_config。1 枚だけ生成されるよう文言で制御する。batch-flow.html の decision）。11-1 時点の登録文言は以下（ユーザー決定）。

現実に存在していそうで、実在はしていないかわいい動物の画像を作ってください。画像には、野生の状態を偶然写真に撮れたかのような架空の動物と、その動物の名前、そして特徴の説明を記載するようにしてください。Instagram に投稿するため、Instagram に合わせたサイズにしてください。生成する画像は1枚のみとしてください。

生成パラメータ（モデル・サイズ・品質等の parameters カラム）は 11-2（画像生成 API の疎通確認）で当たりを付けてから設定する。11-1 時点では NULL。

5. キャプションの作り方

未定（12-1 で caption_templates を登録して確定する）。画像内に動物名・特徴の説明を焼き込む仕様のため、キャプション本文は補足的な固定文 + ハッシュタグを想定。

6. 失敗時ポリシー

共通の失敗時ポリシー（operation.html セクション 4）に準拠。セット固有の追加ポリシーはなし。

7. 共通フローからの差分・特記事項

共通フロー（batch-flow.html）どおりで差分なし。名前・特徴のテキストを画像に合成する処理はプロンプト側で完結させ、共通骨格・DB スキーマの変更は不要。初期方式（gpt-image-single）の方式別設計書は本採用後に docs/app/generators/gpt-image-single.html として作成する（11-4）。
