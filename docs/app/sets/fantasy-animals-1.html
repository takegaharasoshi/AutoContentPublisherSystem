セット別設計書: 架空のかわいい動物図鑑

作成中（13-1 でスケジュール確定済み。13-2 で最終化） set_code: fantasy-animals-1 最終更新: 2026-07-22
本書の位置づけ 本セット固有の設定・運用情報を記録する（11-1 で作成）。共通の処理フロー・データモデル・運用ルールは batch-flow.html / data-model.html / operation.html を参照し、本書には差分のみ書く（design-outline.html セクション 1.1）。

1. テーマ

現実に存在していそうで実在しない、架空のかわいい動物を紹介する画像を生成する。野生の姿を偶然写真に収めたかのような自然な写真風の構図とし、画像内に架空の動物の名前とその特徴（外見・生態など）の説明文を記載する。

2. 投稿先アカウント構成

Instagram プロアカウント 1 件（初期スコープどおり 1 セット 1 アカウント）。12-1 で sns_accounts と SNS 認証 Secret を登録して確定した。

アカウント準備（Instagram プロアカウント化・Facebook ページ連携・Meta アプリ作成・長期アクセストークン取得）の具体手順は operation.html セクション 5.1 に一般化して記録した。

3. スケジュール（投稿頻度）

1 日 3 回（7:00 / 12:00 / 21:00 JST）。Phase 13-1（2026-07-22、ユーザー決定）で確定し、既存プレースホルダ Scheduler（acps-prod-image-generation-schedule）を転用して cron 式 cron(0 7,12,21 * * ? *)（Asia/Tokyo）+ ENABLED で本番化した（operation.html セクション 2.1 手順 3 の decision、現行値は infra/workflow.html セクション 1.5）。画像生成に数分かかるため、実投稿はスケジュール時刻の数分後になる。

4. プロンプトの作り方

固定文言方式。prompt_configs.prompt_text への直接 INSERT で管理する（初期運用は 1 セット 1 prompt_config。1 枚だけ生成されるよう文言で制御する。batch-flow.html の decision）。11-1 時点の登録文言は以下（ユーザー決定）。

現実に存在していそうで、実在はしていないかわいい動物の画像を作ってください。画像には、野生の状態を偶然写真に撮れたかのような架空の動物と、その動物の名前、そして特徴の説明を記載するようにしてください。Instagram に投稿するため、Instagram に合わせたサイズにしてください。生成する画像は1枚のみとしてください。

生成パラメータ（モデル・サイズ・品質等の parameters カラム）は 11-2（画像生成 API の疎通確認）で当たりを付けてから設定する。11-1 時点では NULL。

5. キャプションの作り方

固定文言方式。caption_templates.template_text への直接 INSERT で管理する（初期運用は 1 セット 1 caption_template）。画像内に動物名・特徴の説明を焼き込む仕様のため、キャプション本文は補足的な固定文 + ハッシュタグとした。12-1 時点の登録文言は以下（ユーザー決定。ローカル・Aurora とも id=1）。

どこかにいそうで、どこにもいない動物たち。
今日も新しい一匹に出会いました🐾

6. 失敗時ポリシー

共通の失敗時ポリシー（operation.html セクション 4）に準拠。セット固有の追加ポリシーはなし。

7. 共通フローからの差分・特記事項

共通フロー（batch-flow.html）どおりで差分なし。名前・特徴のテキストを画像に合成する処理はプロンプト側で完結させ、共通骨格・DB スキーマの変更は不要。初期方式（gpt-image-single）は 11-4 で実装済み。方式別設計書 docs/app/generators/gpt-image-single.html は「方式設計書はセットが本採用した時点で作成」ルール（design-outline.html セクション 1.1）に従い、本採用に伴う設計書整備を行う 13-2 で作成する。
